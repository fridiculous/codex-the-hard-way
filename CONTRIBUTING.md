# Contributing

Codex the Hard Way is a source-driven guide. Contributions should make the harness more understandable by tying explanations to observable behavior and source code.

## Good Contributions

- Add a lab with a concrete checkpoint.
- Update a lab against a new source checkout and record the commit.
- Replace vague language with a source-backed explanation.
- Add a diagram that names real components rather than generic boxes.
- Add an exercise that forces the reader to inspect a boundary.

## Standards

- Prefer exact source links, commit hashes, and search queries.
- Avoid claims that cannot be verified from source or runtime behavior.
- Keep labs small enough that a focused reader can complete one in a sitting.
- Explain policy and security boundaries carefully.
- Mark speculative conclusions clearly.

## Writing Style

- Use direct, plain language.
- Favor numbered steps and checkpoints.
- Avoid marketing language.
- Do not assume Codex internals are stable across versions.
- When in doubt, show the source probe that led you to the conclusion.
